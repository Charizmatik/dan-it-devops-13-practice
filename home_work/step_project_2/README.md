# Завдання

Створіть тестовий репозиторій на GitHub із застосунком Node.js із `forStep2`.

Створіть тестовий обліковий запис на Docker Hub (безкоштовно): Docker Hub.

Використовуючи Vagrant, створіть дві віртуальні машини (VM): перша для сервера Jenkins та друга для виконавця Jenkins (Jenkins worker).

Встановіть Docker і запустіть Jenkins на першій VM, вручну або через `Vagrantfile`.

Вручну або за допомогою `Vagrantfile` додайте встановлення Docker та Jenkins worker безпосередньо на другу віртуальну машину (VM) без використання Docker для Jenkins.

Підключіть Jenkins worker до головного вузла (master node). Переконайтеся, що можна запустити тестовий pipeline на Jenkins worker.

Додайте credentials у Jenkins зі своїм логіном і паролем Docker Hub.

Створіть тестовий pipeline, використовуючи Groovy. Pipeline має запускатися вручну та виконувати такі кроки:

1. Отримати код (Pull the code).
2. Зібрати Docker-образ на Jenkins worker.
3. Запустити Docker-образ із тестами.
4. Якщо тести пройшли успішно, Jenkins має увійти у Docker Hub за допомогою збережених credentials з кроку 7 та відправити (push) зібраний образ у Docker Hub.

Якщо тести провалилися, вивести повідомлення: `Tests failed`

# Формат здачі

1. Код з Node.js app у вас в репозиторії
2. Скрін вашого створеного акаунту на Docker Hub
3. Код `Vagrantfile` або скріни встановлення Docker й Jenkins на першу віртуалку
4. Так само для другої віртуалки - код або скріншоти.
5. Скріншоти підключення Jenkins worker, додавання credentials вашого Docker Hub.
6. Код пайплайну
7. Скріни роботи пайплайну (покроково)
