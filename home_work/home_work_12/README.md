## REST API
### ЧАСТИНА 1: Розробити REST API
#### Функціональність
Розробіть REST API для керування студентами мовою Python з використанням бібліотеки Flask у файлі app.py, із збереженням даних у файлі students.csv. API має підтримувати запити GET, POST, PUT, PATCH та DELETE. Кожен студент повинен мати поля id, імʼя, прізвище та вік.

#### HTTP-запити
#### GET
- Отримати інформацію про конкретного студента (відобразити всі наявні дані):
- За його ID. Якщо переданий ID не знайдено у CSV-файлі, повернути помилку.
- За його прізвищем. Якщо є кілька студентів з однаковим прізвищем, відобразити всіх; якщо передане прізвище не знайдено у CSV-файлі, повернути помилку.
- Отримати список усіх студентів (відобразити всі наявні дані).
#### POST
- Створити нового студента.
- Поле ID має генеруватися автоматично з інкрементом +1.
- Тіло запиту POST повинно містити імʼя, прізвище та вік.
- Якщо у тілі POST передано неіснуюче поле або не передано жодного поля, повернути помилку.
- Якщо будь-яке з обов’язкових полів відсутнє у тілі POST, повернути помилку без запису у CSV-файл.
- У разі успішного запиту повернути інформацію про доданого студента.
#### PUT
- Оновити інформацію про студента за його ID.
- Має бути можливим оновити поля імʼя, прізвище та вік.
- Якщо переданий ID не знайдено у CSV-файлі, повернути помилку.
- Тіло запиту PUT повинно містити імʼя, прізвище та вік відповідно.
- Якщо у тілі PUT передано неіснуюче поле або не передано жодного поля, повернути помилку.
- У разі успішного запиту повернути оновлену інформацію про студента.
#### PATCH
- Оновити вік студента за його ID.
- Має бути можливим оновити поле "вік".
- Якщо переданий ID не знайдено у CSV-файлі, повернути помилку.
- Тіло запиту PATCH повинно містити вік.
- Якщо у тілі PATCH передано неіснуюче поле або не передано жодного поля, повернути помилку.
- У разі успішного запиту повернути оновлену інформацію про студента.
#### DELETE
- Видалити студента з CSV-файлу за його ID.
- Якщо переданий ID не знайдено у CSV-файлі, повернути помилку.
- У разі успішного запиту повернути повідомлення, що підтверджує успішне видалення студента.

### ЧАСТИНА 2: Створити test_requests.py
#### Функціональність
Створіть файл test_requests.py для перевірки створеного REST API. У цьому файлі продемонструйте роботу всіх методів з використанням бібліотеки requests у такій послідовності:

1. Отримати всіх наявних студентів (GET).
2. Створити трьох студентів (POST).
3. Отримати інформацію про всіх наявних студентів (GET).
4. Оновити вік другого студента (PATCH).
5. Отримати інформацію про другого студента (GET).
6. Оновити імʼя, прізвище та вік третього студента (PUT).
7. Отримати інформацію про третього студента (GET).
8. Отримати всіх наявних студентів (GET).
9. Видалити першого користувача (DELETE).
10. Отримати всіх наявних студентів (GET).
Відобразити результати виконання у консолі та записати їх до файлу results.txt.

У підсумку слід завантажити (push) файли app.py, test_requests.py, results.txt, requirements.txt та скріншоти виконання до GitHub.
