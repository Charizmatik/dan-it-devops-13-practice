# Розгортання зразкового застосунку Spring PetClinic локально за допомогою Vagrant
Використовуйте вихідний код з [repo](https://gitlab.com/dan-it/groups/devops_soft/-/tree/main/forStep1/PetClinic?ref_type=heads)

Створіть репозиторій на GitLab і зафіксуйте (commit) усі зміни у свій репозиторій GitLab.

Створіть скрипт розгортання для застосунку PetClinic. Використайте Vagrant для автоматизації процесу створення інфраструктури для розгортання за допомогою VirtualBox (бажано). Для provisioning можна використовувати bash, python або ansible — в будь-якій комбінації.

## Підзавдання I – Інфраструктура

Опишіть дві віртуальні машини використовуючи Vagrantfile, одна для застосунку (кодова назва: APP_VM) та для бази даних (кодова назва: DB_VM).
Віртуальні машини повинні працювати під CentOS або Ubuntu.
Якщо не використовується приватна мережа, тоді APP_VM повинна мати проброс порту 8080 на хост-машину.

## Підзавдання II – База даних

Використайте будь-який скрипт налаштування (provisioning script), щоб встановити MySQL і всі необхідні залежності на DB_VM.
Налаштуйте базу даних MySQL так, щоб вона приймала з’єднання лише з приватної підмережі Vagrant.
Створіть нерутового користувача та пароль (кодові назви: DB_USER і DB_PASS) у MySQL. Використайте змінні середовища хоста (environment variable), щоб задати ці значення та передати їх у Vagrantfile через ENV.
Створіть базу даних в MySQL (кодова назва: DB_NAME) і надайте користувачу DB_USER усі привілеї для доступу до цієї бази.

## Підзавдання III – Застосунок

Створіть нерутового користувача (кодова назва: APP_USER), який використовуватиметься для запуску застосунку на APP_VM.
Використайте будь-який provisioner, щоб встановити Java JDK, Git та всі необхідні залежності на APP_VM.
Клонуйте цей репозиторій у робочу теку (кодова назва: PROJECT_DIR).
Використайте інструмент Maven для виконання тестів і створення пакету застосунку. Докладніше дивіться у 5-хвилинній документації Maven. Для зручності в проєктній теці вже є Maven Wrapper (mvnw), який автоматично завантажує і запускає потрібну версію Maven.
Якщо тестування і пакування пройшло успішно, отримайте файл *.jar з теки $PROJECT_DIR/target і помістіть його в домашню теку користувача APP_USER (кодова назва: APP_DIR).
Встановіть змінні середовища (environment variables) у APP_VM (бажано використовувати ті ж, що передавались через ENV у DB_VM):

DB_HOST - IP-адреса DB_VM
DB_PORT - порт MySQL (типово 3306)
DB_NAME - назва бази даних MySQL
DB_USER - користувач MySQL
DB_PASS - пароль користувача MySQL
Запустіть застосунок від імені користувача APP_USER за допомогою команди java -jar 

Якщо все виконано успішно, застосунок PetClinic буде доступний за адресою $APP_VM_IP:8080

## Формат здачі:

1. Код змісту Vagrantfile

2. Скрін вашого git репозиторія з проектом PetClinic

3. Скрін підключення з однієї віртуалки до бази даних, яка на іншій

4. Якщо якісь кроки виконувались не в Vagrantfile, а вручну, то скріни тих кроків

5. Скрін роботи аплікації на 8080 порту з прикладом додавання своїх даних

6. Скрін сайту з існуючими й своїми доданими даними
