# Dockerfile

## Частина 1

Створіть `Dockerfile` для REST API із HOMEWORK 12. Необхідно використати `gunicorn` як вебсервер для запуску API та налаштувати його на роботу за адресою `0.0.0.0:8000`.

Базовий образ (`base image`) у `Dockerfile` може бути будь-який, на ваш вибір.

## Частина 2

Створіть образ (`image`) на основі вашого `Dockerfile`, запустіть контейнер за допомогою команди `docker run` та продемонструйте роботу створеного образу, виконавши запит (`GET`, `POST` або `PUT` — на ваш вибір) до контейнера через Postman.

У результаті до репозиторію GitHub потрібно завантажити ваш REST API, `requirements.txt`, `Dockerfile`, `.dockerignore` та скріншоти запитів у Postman.

## Формат здачі ДЗ

1. Код `Dockerfile`
2. Скрін запуску контейнера
3. Скріншоти запитів `GET`, `POST` або `PUT` (на вибір) за допомогою Postman
4. Код `requirements.txt`, `Dockerfile`, `.dockerignore`
